# 安装 GitLab

  1. 拉取 GitLab 镜像 gitlab-ce 为稳定版本，默认为最新版本
  1. 运行 gitlab 镜像
- -d：后台运行
- -p：将容器内部端口向外映射
- --name 命名容器名称
- -v：将容器内数据文件夹或者日志、配置等文件夹挂载到宿主机指定目录

运行成功后出现一串字符串，为容器 ID

# 配置 GitLab

  1. 获取本地 IP 地址，windows 用 ipconfig 命令，linux 用 ifconfig 命令

  2. 按上面的方式，gitlab 容器运行没问题，但在 gitlab 上创建项目的时候，生成项目的 URL 访问地址是按容器的 hostname 来生成的，也就是容器的 id。作为 gitlab 服务器，我们需要一个固定的 URL 访问地址，于是需要配置 gitlab.rb（宿主机路径：/home/gitlab/config/gitlab.rb）

  1. 修改 registry_external_url, 保存
  1. 重启容器

# 重置管理员密码

  1. 进入 gitlab 容器，gitlab（容器名字）
  1. 进入控制台
  1. 等待控制台加载完毕，寻找 root 用户
  1. 修改密码
  1. 保存到数据库
